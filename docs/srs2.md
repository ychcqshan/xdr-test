## 一、文档概述

### 1.1 产品名称
网络安全XDR（扩展检测与响应）平台

### 1.2 产品定位
面向企业级市场的终端安全监控与响应系统，专为满足**国产化环境**与**高安全要求**场景设计，支持Windows和国产化操作系统双平台。

### 1.3 产品愿景
构建"采集-分析-响应-优化"的闭环安全运营体系，实现从被动防御到主动预测的安全运营转型。

## 三、核心功能需求

### 3.1 终端安全监控节点（Agent）

#### 3.1.1 系统基础能力
- **平台兼容性**：
  - Windows：Win7/10/11, Server 2012-2022
  - 国产化OS：麒麟 V10+、统信UOS V20+、欧拉 openEuler
  - CPU架构：x86_64、ARM64、LoongArch
- **轻量化设计**：
  - 安装包体积 ≤50MB
  - 常驻内存 ≤100MB
  - CPU占用率 ≤5%（平均）
- **自保护机制**：
  - 防卸载：内核级保护，需管理员权限才能卸载
  - 防篡改：关键文件完整性校验（SHA-256）
  - 通信加密：TLS 1.3+，双向证书认证

#### 3.1.2 威胁检测与响应能力
- **基线应用**：
  - 接收后端下发的基线配置（JSON格式）
  - 按基线类型（进程/端口/USB/登录/软件）进行实时比对
- **异常检测**：
  - **新增项目**：基线中不存在，当前系统存在
  - **缺失项目**：基线中存在，当前系统不存在
  - **修改项目**：基线中存在，当前系统值发生变化
- **场景化检测**：
  - **勒索软件防御**：
    - **诱饵文件 (Honeyfiles)**：在敏感目录部署 decoy 文件，监控 `ReadDirectoryChangesW` 事件。
    - **[REMOVE] 熵值分析**：本阶段暂不实现。
  - **横向移动检测**：识别 PsExec、WMI 等远程执行工具的滥用；监控 LSASS 进程访问。
  - **无文件攻击识别**：
    - **CMD/指令监控**：
      - Windows: 订阅 Security Event ID 4688 (需开启命令行包含策略)。
      - Linux: 监控 Auditd `execve` 日志，实现比 `history` 更全面的指令捕获。
- **数据采集增强**：
  - **流量采集**：Agent 支持开启**网卡混杂模式**，采集全量网络通信元数据，辅助后端拓扑刻画。
- **响应动作**：
  - 自动终止恶意进程
  - 阻断异常网络连接
  - 隔离违规USB设备
  - 生成攻击时间线供后台分析

#### 3.1.3 数据管理与通信能力
- **数据采集**：
  - 采集维度：进程、网络、USB、登录、资产
  - 采集频率：可配置（默认60秒）
  - 采集内容：关键行为数据+元数据
- **数据压缩与加密**：
  - 采用GZIP压缩日志数据（压缩率≥70%）
  - TLS 1.3+加密传输，双向证书认证
  - 支持断网续传（本地缓存≥7天）
- **分级上报机制**：
  - **高危事件**：实时上报（延迟<1秒），全量日志
  - **中危事件**：每5分钟上报，摘要日志
  - **低危事件**：每30分钟上报，摘要日志
- **心跳与状态同步**：
  - 每5分钟上报Agent状态（在线/离线、版本号、资源占用）
  - 异常状态（如防篡改触发）实时上报

#### 3.1.4 用户信息管理（新增）
- **初次安装配置**：
  - 首次安装时弹出配置窗口（Windows平台），国产化系统通过后台配置
  - 收集基础信息：姓名、部门、单位名称、手机号、邮箱
  - 信息加密存储（AES-256）
- **基础管理功能**：
  - 查看当前用户信息
  - 修改联系方式（需管理员权限）
  - 重置Agent配置（恢复默认设置）
- **信息上报**：
  - 首次安装时自动上报用户信息至后端
  - 用户信息与Agent绑定，用于安全事件关联分析
- **扩展属性搜索**：
  - 全站支持按 “所属单位” 和 “责任人” 进行跨维度告警、资产及基线检索。

#### 3.1.5 合规与扩展能力
- **等保合规支持**：
  - 内置等保2.0、PCI DSS基线检查模板
  - 自动扫描弱口令、未授权访问、高危端口
  - 一键生成合规报告（PDF/Excel格式）
- **第三方集成**：
  - 提供SDK支持与EDR、SIEM系统联动
  - 开放API供SOAR平台调用响应动作
  - 预置集成模板（Splunk、QRadar、阿里云安全中心）
- **日志审计**：
  - 记录Agent自身操作日志（策略更新、升级记录）
  - 支持导出与后台集中分析
  - 日志包含：操作时间、操作人、操作内容、结果

### 3.2 安全运营中心（后端服务）

#### 3.2.1 基线与资产管理服务
- **基线学习引擎**：
  - **快速学习**：1小时窗口（测试环境）
  - **标准学习**：7天窗口（生产环境，默认值）
  - **自定义学习**：1-30天灵活配置
- **资产状态聚合 (State Aggregation)**：
  - **端云协同**：Agent 上报 `ADD/REMOVE` 增量或 `FULL` 全量指令。
  - **后端状态机**：由 `asset-service` 维护资产的活跃状态 (`ACTIVE/INACTIVE`)，确保跨会话的数据一致性。
  - **指纹比对**：通过 `pid + createTime` (进程) 或 `laddr + raddr` (连接) 等唯一指纹进行精确状态锁定。
- **时光机回溯 (Time Machine)**：
  - **时序快照**：支持加载主机在历史任意时刻的资产切片。
  - **可视化穿越**：前端提供时间轴控件，进入历史模式后可查看当时真实的进程、网络及软件状态，并伴有显著的视觉提醒。
- **基线创建方式**：
  - 学习创建：通过学习期自动收集数据
  - 导入创建：导入当前系统数据作为基线
  - 复制创建：从其他Agent复制基线配置
  - 手动创建：管理员手动添加基线项目
- **基线类型**：
  - 进程基线（PROCESS）
  - 端口基线（PORT）
  - USB设备基线（USB）
  - 登录基线（LOGIN）
  - 软件基线（SOFTWARE）
### 3.2 接口与功能详情 (Phase 2 已实现)
- **高级检测模块**：
  - **Windows (Implemented)**: 基于 Security Event ID 4688 的命令行采集与匹配。
  - **Linux (Implemented)**: 基于 Auditd 日志的执行指令流捕获。
  - **勒索检测 (Implemented)**: 基于 Honeyfiles (watchdog 监听) 的触发机制。
  - **资产画像 (Implemented)**: CPU架构/核心、软件清单、USB插拔审计、登录历史、流量拓扑。
- **采集频率**：支持按需配置。

#### 3.2.2 威胁分析与告警服务
- **告警级别**：
  - 严重（CRITICAL）：红色
  - 高危（HIGH）：橙色
  - 中等（MEDIUM）：黄色
  - 低危（LOW）：蓝色
- **告警状态**：
  - 待处理（NEW）
  - 已确认（ACKNOWLEDGED）
  - 已解决（RESOLVED）
  - 已忽略（IGNORED）
- **告警关联分析**：
  - 事件聚合：将相关事件聚合成攻击链
  - 威胁溯源：生成攻击时间线和路径图
  - 优先级计算：基于威胁等级、资产价值计算告警优先级
- **自动化响应**：
  - 预置响应动作库（进程终止、网络阻断、USB隔离）
  - 基于策略的自动化响应
  - 人工确认后自动执行

#### 3.2.3 策略管理服务
- **策略配置**：
  - 按资产组或单个主机配置检测策略
  - 调整检测敏感度（高/中/低）
  - 启用/禁用特定检测模块
- **策略模板**：
  - 预置策略模板（办公环境、服务器、数据库）
  - 自定义策略模板
  - 策略版本管理
- **策略下发**：
  - 实时下发策略变更
  - 灰度发布策略
  - 策略回滚机制

#### 3.2.4 远程升级服务
- **版本管理**：
  - 支持Agent版本批量升级或灰度升级
  - 升级包签名验证，防止中间人攻击
- **升级状态**：
  - 实时查看各Agent升级进度与状态
  - 支持回滚到上一版本
  - 升级失败自动告警

#### 3.2.5 合规管理服务（新增）
- **合规检查**：
  - 内置等保2.0、PCI DSS、GDPR等合规标准
  - 自动扫描合规项（弱口令、未授权访问、高危端口等）
  - 生成合规差距分析报告
- **合规基线管理**：
  - 创建自定义合规基线
  - 关联资产与合规要求
  - 设置合规检查频率（实时/每日/每周）
- **合规报告**：
  - 一键生成合规报告（PDF/Excel/CSV）
  - 报告包含：合规项状态、差距分析、改进建议
  - 支持按部门/资产组生成差异化报告
- **合规审计**：
  - 记录合规检查历史
  - 支持合规检查结果追溯
  - 自动提醒即将到期的合规检查

#### 3.2.6 前端界面服务（新增）
- **系统导航**：
  - 顶部导航栏：仪表盘、资产、威胁、合规、策略、升级
  - 侧边栏：资产分组管理、告警分类、用户管理
- **核心界面**：
  - **仪表盘**：
    - **视觉风格**：采用 **Linear.app** 极简设计系统，深色模式高对比度显示 (#0a0a0b)。
    - **布局方案**：使用 **Bento Grid** (便当盒) 布局实现响应式模块化排布。
    - **核心组件**：
      - **全域威胁地图**：可视化动态拓扑图谱。
      - **高危告警流水**：实时滚动卡片。
      - **系统健康度微图**：使用 Sparklines 展示资源负载趋势。
  - **资产视图**：
    - 资产列表（按部门/组/类型分组）
    - 资产详情（硬件信息、软件列表、用户信息、进程/连接分页展示）
    - 资产风险评分（基于基线偏离度、威胁历史）
  - **威胁视图**：
    - 告警列表（按状态/级别/时间筛选）
    - 威胁详情（攻击链分析、时间线、响应记录）
    - 威胁溯源（横向移动路径图）
  - **合规视图**：
    - 合规状态概览（合规率、高风险项）
    - 合规检查报告（可导出）
    - 合规差距分析（按项展示差距）
  - **策略视图**：
    - 策略模板库
    - 策略配置（按资产组应用）
    - 策略版本历史
  - **升级视图**：
    - Agent版本分布
    - 升级计划（灰度发布）
    - 升级状态监控
- **用户界面特性**：
  - 响应式设计（适配PC/平板）
  - 多语言支持（中文/英文）
  - 操作日志审计（记录用户操作）
  - 个性化配置（自定义仪表盘）

## 六、设计亮点

1. **终端-后端协同架构**：终端Agent专注于数据采集与轻量级威胁检测，后端服务专注于基线管理、威胁分析与响应，形成高效协同架构。

2. **场景化威胁检测**：针对勒索软件、横向移动、无文件攻击等典型攻击场景提供深度检测能力，实现从"被动响应"到"主动防御"的转变。

3. **合规驱动设计**：内置等保2.0合规检查模板，自动生成合规报告，满足政府和企业合规审计需求。

4. **用户信息管理闭环**：终端首次安装收集用户信息，与安全事件关联，提升安全事件分析的精准度。

5. **专业合规管理**：安全运营中心提供完整的合规管理功能，包括合规检查、差距分析、报告生成。

6. **企业级前端界面**：提供专业、易用的前端界面，支持多维度安全态势展示和操作。

7. **国产化深度适配**：明确支持麒麟、统信UOS、欧拉等国产操作系统及飞腾、龙芯等国产CPU，满足信创环境需求。

8. **轻量化与高性能平衡**：终端Agent资源占用极低（内存≤100MB，CPU≤5%），同时支持高并发（>1000 Agent）。

## 七、术语表

| 术语 | 英文 | 定义 |
|------|------|------|
| Agent | Agent | 部署在终端的监控程序 |
| 基线 | Baseline | 系统正常运行时的标准状态数据 |
| 异常 | Anomaly | 偏离基线的行为或状态 |
| 告警 | Alert | 检测到异常时生成的安全通知 |
| 威胁链 | Threat Chain | 从攻击初始到最终影响的完整攻击路径 |
| 横向移动 | Lateral Movement | 攻击者在内网中从一个系统移动到另一个系统的攻击行为 |
| 无文件攻击 | Fileless Attack | 不使用文件执行恶意代码的攻击方式 |
| 合规差距 | Compliance Gap | 实际状态与合规要求的差异 |
