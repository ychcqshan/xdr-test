# XDR Phase 2: 高级检测与增强采集 (Completed)

## 概述
Phase 2 的核心目标是在 Phase 1 MVP 的基础上，提升终端数据采集的深度，并引入针对勒索软件、无文件攻击以及横向移动的专业检测能力。同时，同步增强后端分析与前端可视化能力，实现对终端状态的精细化评估。

## 用户审核项 (User Review Required)

> [!IMPORTANT]
> - **检测技术选型**：为了兼顾性能与深度，终端将引入 **ETW (Windows)** 和 **Auditd (Linux)** 作为指令监控的核心。对于 **CMD** 指令，将通过开启 Windows 安全审计 **Event ID 4688** 并启用命令行包含策略来捕获。
> - **基础设施调整**：根据反馈，Phase 2 将**推迟** Kafka 和 Elasticsearch 的引入，继续沿用现有的数据库架构，优先保证功能实现。
> - **响应策略**：初期所有自动化响应动作默认为 "手动确认" 模式，防止因误报导致业务中断。

## 拟定变更

### 1. 终端 Agent (xdr-agent) 采集增强

#### [NEW] 全维度采集字段映射
Agent 将扩展以下采集维度：
- **进程信息**：PID, 进程名, CPU/内存占用, 状态, 启动时间, 所属用户。
- **端口信息**：端口号,协议, 状态, 关联 PID/进程名。
- **主机信息**：CPU (品牌/架构/核心/频率), 内存 (总量/可用/使用率), 存储设备, OS 信息, MAC/IP 地址。
- **软件与 USB**：已安装软件列表 (名称/版本/路径), USB 设备 (名称/VID/PID/厂商)。
- **网络与登录**：接口详细配置, 登录日志 (用户名/类型/时间/来源 IP/状态)。
- **流量监控**：采集终端基础流量数据（源/目的 IP、端口、报文数），为后端拓扑刻画提供基础。

#### [NEW] 高级检测模块
- **勒索软件监控**：
  - **诱饵文件 (Honeyfiles)**：在敏感目录部署 decoy 文件，监控 `ReadDirectoryChangesW` 事件。
  - **[DELETED] 熵值分析**：按需求反馈，本阶段暂不实现熵值分析功能。
- **指令执行监控**：
  - **Windows (CMD/PowerShell)**：
    *   **实现机制**：订阅 **Security Event ID 4688**。需开启“在进程创建事件中包含命令行”审计策略。
    *   **提取逻辑**：Agent 实时解析 `Process Command Line` 字段获取完整参数。
  - **Linux (Auditd)**：利用 `auditd` 监控 `execve` 系统调用，尾部随动读取 `/var/log/audit/audit.log` 拼接完整命令流。比 `history` 更全面且不可篡改。
- **横向移动检测**：
  - 匹配高频攻击特征（如 `PsExec` 服务创建、`WMI` 远程进程调用、`Mimikatz` 的 LSASS 访问特征）。

### 2. 后端服务 (xdr-server) 功能完善

#### [MODIFY] 业务逻辑增强
- **详情与统计**：增加针对扩展采集字段的管理接口，支持按 Agent 查看资产快照。
- **访问关系刻画 (Topology)**：
  - **采集策略**：Agent 开启**网卡混杂模式 (Promiscuous Mode)** 采集原始流量信息。
  - **后端逻辑**：根据流量数据自动计算并存储资产间的访问拓扑。
- **基线管理功能丰富**：
  - **基线模板化**：支持将“洁净主机”快照保存为模板。
  - **漂移分析 (Drift Analysis)**：后端自动对比基线与当前状态，记录偏离项。
  - **版本管理**：支持基线历史版本的对比与回滚。

### 手动验证
- **UI 审计**：确认详情页及 Visual Diff 界面的交互逻辑符合设计预期。
- **拓扑验证**：确认网络访问行为能正确转化为后端拓扑链路。
