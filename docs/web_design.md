# XDR 平台前端设计规范 (Web Design Specification)

本规范定义了 XDR 平台的视觉风格、交互准则及组件标准，旨在确保全站 UI 呈现出 **Fintech Elite (金融级精英)** 的高级质感与 **Liquid Glass (流体玻璃)** 的通透空间感。

---

## 🎨 1. 视觉系统 (Visual System)

### 1.1 核心调色盘
| 角色 | 色值 (HEX) | 用途 |
| :--- | :--- | :--- |
| **背景基色** | `#0a0a0b` | 深色模式核心背景 |
| **卡片背景** | `rgba(255, 255, 255, 0.03)` | 玻璃拟态卡片，配合 1px 描边 |
| **主色 (Primary)** | `#3b82f6` | 极客蓝，用于主按钮与关键态势 |
| **危险 (Danger)** | `#ef4444` | 用于 CRITICAL 告警 |
| **警告 (Warning)**| `#f59e0b` | 用于 HIGH 告警与“时间机器”模式 |
| **成功 (Success)**| `#10b981` | 用于在线状态 / 已解决告警 |

### 1.2 空间与布局
- **核心布局**: **Bento Grid (便当盒布局)**。使用非对称网格划分功能区，利用不规则的间距引导视觉重心。
- **圆角标准**: 组件统一采用 `12px` (Large) 圆角，按钮及微缩组件采用 `8px` (Medium)。
- **阴影/描边**: 杜绝重阴影，采用 `1px solid rgba(255,255,255,0.1)` 作为组件分界。

## ⚡ 2. 交互准则 (Interaction Principles)

### 2.1 动效哲学 (Motion)
- **拒绝过度反馈**: 移除了列表行在 Hover 时的 `translateY` (向上浮出) 处理，避免在快速操作时产生视觉疲劳。
- **微动效**: 仅保留背景色加深及容器描边高亮（Border-flash）效果。
- **渐入排队**: 列表加载时，利用 Vue 的 `transition-group` 配合 `stagger-delay` 实现条目有序滑入。

### 2.2 信息密度控制
- **搜索表单**: 
  - 标签 (Label) 字体设为 `13px`。
  - 采用水平分布或紧凑排列，确保搜索栏不占用超过 12% 的首屏高度。
- **表格溢出处理**: 
  - 对于 Hostname、Agent ID 等长字段，采用 `el-popover`。
  - **交互**: 鼠标滑过时气泡显示全量信息，表格内保持单行窄边，确保数据对比的紧凑性。

## 🧩 3. 核心组件规范 (Key Components)

### 3.1 高性能详情分页
- **适用场景**: 资产详情页下的进程、网络连接、登录审计等列表。
- **标准**:
  - **每页记录数**: 12 条。
  - **位置**: 卡片底部右侧，采用微缩型分页器 (Small size)。
  - **逻辑**: 优先采用前端分页以降低后端查询压力（针对单次 API 返回的聚合快照）。

### 3.2 仪表盘态势看板
- **可视化**: 采用 ECharts 暗色主题，线条宽度保持在 `1.5px` 以下，配合 `AreaStyle` 面容渐变。
- **实时性**: WebSocket 推送触发局部数据响应式更新，而非整页刷新。

### 3.3 时间机器 (Time Machine)
- **激活态**: 顶部出现 `Warning` 级别的通知条 (Fixed Banner)。
- **视觉反馈**: 详情页 Tabs 区域色彩饱和度降低，暗示“历史快照”模式。

## 🏗️ 5. 界面功能组成 (Interface Functional Composition)

### 5.1 核心导航架构
- **AppLayout**: 采用左侧固定导航 + 顶部全局操作栏 + 中央响应式视图。
- **面包屑导航**: 实时反应路由层级，支持快速回溯。
- **全局搜索**: 顶部操作栏集成跨模块搜索入口（后续支持）。

### 5.2 功能模块分类
| 模块类型 | 包含组件 | 核心价值 |
| :--- | :--- | :--- |
| **态势感知** | Dashboard, Timeline | 宏观安全态势全局把控 |
| **资产生命周期** | AssetList, AssetDetail | 终端档案管理与画像聚合 |
| **威胁治理** | AlertList, AlertDetail | 安全事件闭环响应与处突 |
| **安全合规** | Compliance, Baseline | 合规基准对齐与持续评估 |
| **运维管理** | Upgrade, Policy, User | 平台治理、策略分发与用户权限 |

## 📖 6. 页面详细描述 (Detailed Page Descriptions)

### 6.1 安全概览 (Dashboard)
- **核心功能**: 汇聚全平台核心安全指标。
- **关键元素**: 
  - **态势图表**: 攻击趋势折线图、风险资产 Top 5。
  - **健康度面板**: 在线 Agent 分布、系统资源负载微缩图。
  - **告警流水线**: 实时滚动展示最新 10 条高危告警。

### 6.2 资产管理 (Asset Management)
- **资产列表 (AssetList)**:
  - **搜索能力**: 支持根据 Hostname、Agent ID 进行搜索，并提供单位 (Unit) 与责任人 (Responsible Person) 过滤。
  - **交互**: 鼠标悬停显示详细 Hostname。
- **资产详情 (AssetDetail)**:
  - **主机画像**: 聚合显示 CPU 核心、内存、网卡、IP 等基础信息。
  - **数据看板**: 进程列表、网络连接、安装软件、USB 设备、登录审计、网络流量。
  - **分页标准**: 列表均采用 **12 条/页** 的前端分页。

### 6.3 威胁告警 (Threat Alerts)
- **告警列表 (AlertList)**:
  - **分级展示**: 采用彩色 Tag (Critical:红, High:橙...)。
  - **处突链路**: 点击列表项可快速进行告警确认、误报忽略或触发预置响应动作。
  - **数据同步**: 与后端同步实时的单位与责任人标签，支持组织维度的威胁追溯。

### 6.4 基线管理 (Baseline Management)
- **核心功能**: 定义和比对终端的稳态配置。
- **主要逻辑**: 
  - **基线概览**: 按类型（进程、端口、USB、软件、登录）分类展示当前生效基线。
  - **基线搜索**: 支持按“单位”和“责任人”检索特定人群或部门的基线状态。

### 6.5 合规管理 (Compliance Management)
- **核心功能**: 自动化合规对齐（等保、行业标准）。
- **主要逻辑**:
  - **合规扫描**: 一键触发全网资产或特定组的合规审计。
  - **结果展示**: 直观展示合规率、未通过项及修复建议。

### 6.6 策略管理 (Policy Management)
- **核心功能**: 下发终端防护策略。
- **功能点**: 采集频率配置、模块开关控制、灰度发布开关。

### 6.7 远程升级 (Upgrade)
- **核心功能**: Agent OTA 升级。
- **功能点**: 版本分布统计、升级任务创建、任务进度追踪。

### 6.8 用户管理 (User Management)
- **核心功能**: 平台管理员账号管理。
- **功能点**: RBAC 角色分配、账户停启用、审计日志（后续实装）。

## 🛠️ 7. 技术栈实现建议
- **CSS**: 严格遵循 Tailwind CSS 工具类，严禁编写 ad-hoc 的行内样式。
- **图标**: 统一使用 `Lucide-Vue-Next`。
- **字体**: 优先调用系统字体（Inter / PingFang SC），Header 部分可采用中性无衬线字体。
